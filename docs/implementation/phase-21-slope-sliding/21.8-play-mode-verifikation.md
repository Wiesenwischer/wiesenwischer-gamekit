# 21.8 Play Mode Verifikation

> Kein eigener Commit/Branch — wird als Teil des letzten Feature-Branches oder direkt auf dem Integration-Branch verifiziert.

---

## Ziel

Alle Sliding-Features im Play Mode manuell testen und bestätigen.

---

## Test-Szene Vorbereitung

In der `AnimationTestScene` (oder einer neuen Test-Szene) folgende Elemente sicherstellen:

1. **Steile Rampe** (> MaxSlopeAngle, z.B. 50-60°)
2. **Moderate Rampe** (< MaxSlopeAngle, z.B. 30°) — als Vergleich
3. **Rampe mit Grenzwinkel** (= MaxSlopeAngle ± 2°) — für Hysterese-Test
4. **Klippe an steilem Hang** — für Falling-Transition
5. **Erhöhte Plattform über steilem Hang** — für Fall→Slide Test

---

## Manuelle Test-Checkliste

### Grundfunktion

- [ ] **Grounded → Slide:** Character auf steiler Rampe (> MaxSlopeAngle) → wechselt automatisch in Slide-State
- [ ] **Slide-Geschwindigkeit:** SlopeSlideSpeed Config-Wert hat sichtbaren Effekt
- [ ] **Steilheitsabhängig:** Steilere Rampe = schnelleres Rutschen (bei `UseSlopeDependentSlideSpeed = true`)
- [ ] **Slide-Animation:** Korrekte Slide-Pose wird abgespielt (kein T-Pose, kein Flackern)
- [ ] **Rotation:** Character dreht sich in Rutsch-Richtung (Hangabwärts)

### Transitions

- [ ] **Slide → Grounded:** Am Ende der Rampe (flacher Boden) → smooth Übergang zu Idle oder Moving
- [ ] **Slide → Falling:** Klippe am Hang → Character fällt
- [ ] **Falling → Slide:** Von Plattform auf steile Rampe fallen → direkt in Slide (nicht HardLand)
- [ ] **Kein Flackern:** Am Grenzwinkel kein Hin-und-Her zwischen States

### Spieler-Steuerung

- [ ] **Lenkung:** Seitlicher Input während Slide → sichtbare seitliche Ablenkung
- [ ] **Jump aus Slide:** Jump-Taste → Character springt (mit reduzierter Kraft)
- [ ] **Jump deaktiviert:** `CanJumpFromSlide = false` → Jump wird ignoriert

### Animation

- [ ] **CrossFade Grounded → Slide:** Sanfter Übergang (~0.2s)
- [ ] **CrossFade Slide → Idle:** Sanfter Übergang (~0.3s, "Aufstehen")
- [ ] **CrossFade Slide → Falling:** Schneller Übergang (~0.15s)
- [ ] **Looping:** Animation loopted korrekt bei langem Hang

### Config-Werte

- [ ] **SlideAcceleration:** Niedriger Wert → langsames Eingleiten, hoher → sofort volle Speed
- [ ] **SlideSteerStrength:** 0 → keine Lenkung, 1 → starke seitliche Kontrolle
- [ ] **SlideExitHysteresis:** Sichtbar kein Flackern bei 3°, Flackern bei 0°
- [ ] **MinSlideTime:** Character bleibt mindestens 0.2s im Slide
- [ ] **SlideJumpForceMultiplier:** Sprung aus Slide ist sichtbar niedriger als normaler Sprung

### Performance

- [ ] **Kein Overhead:** Normale Grounded-Bewegung (flacher Boden) hat keinen spürbaren Performance-Einbruch
- [ ] **Kein GC:** Profiler zeigt keine Allocations im Sliding-Path

## Bekannte Einschränkungen

- Keine VFX/Particles (späteres Feature)
- Keine Sound-Effekte (späteres Feature)
- Keine Netzwerk-Synchronisation (Phase 6/7)
- IK (FootIK) wird während Sliding ggf. nicht deaktiviert — visuell prüfen ob nötig
