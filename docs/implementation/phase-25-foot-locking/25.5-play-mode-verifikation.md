## Ziel

Manuelle Verifikation im Play Mode. Visuelles Prüfen, dass Foot Sliding bei Animations-Übergängen eliminiert ist und keine Regression bei bestehenden Features auftritt.

## Test-Szenarien

### Foot Locking Kern-Funktionalität

- [ ] **Walk → Idle:** Beim Stehenbleiben gleiten die Füße NICHT zusammen. Füße bleiben an der Position des letzten Schritts.
- [ ] **Run → Stop:** Füße bleiben am Ort, Stop-Animation spielt mit geplanteten Füßen.
- [ ] **Idle auf flachem Boden:** Füße stabil am Ort, kein Micro-Wackeln.
- [ ] **Drehen im Stand:** Füße bleiben relativ zum Character (drehen mit), kein Gleiten.

### Kein Lock bei Bewegung

- [ ] **Walk/Run:** FootLock NICHT aktiv während der Fortbewegung. Füße folgen der Walk/Run-Animation.
- [ ] **Schnelles Umkehren:** Kein unnatürliches Festkleben der Füße beim Richtungswechsel.

### Koordination mit FootIK

- [ ] **Walk auf Terrain-Stufen:** FootIK Terrain-Anpassung funktioniert (FootLock nicht aktiv, Füße passen sich dem Boden an).
- [ ] **Idle auf unebenem Terrain:** FootIK + FootLock arbeiten zusammen — gelockte Füße stehen korrekt auf dem Boden.

### Airborne-Verhalten

- [ ] **Sprung:** Während des Sprungs kein FootLock aktiv. Nach der Landung: Füße werden korrekt neu gelockt.
- [ ] **Fallen:** Kein FootLock während des Fallens.

### Regression

- [ ] **Bestehende FootIK-Funktionalität:** Terrain-Anpassung funktioniert wie bisher.
- [ ] **Bestehende LookAtIK:** Keine Änderung am Blick-Verhalten.
- [ ] **Crouching:** Crouch-Idle hat stabile Füße.
- [ ] **Knie:** Keine durchgedrückten Knie auf flachem Boden (Phase 24 Fix intakt).

### Edge Cases

- [ ] **FootLock im Inspector deaktivieren:** Keine Fehlermeldungen, FootIK übernimmt.
- [ ] **IKManager Master-Weight = 0:** Keine IK aktiv, keine Fehler.
