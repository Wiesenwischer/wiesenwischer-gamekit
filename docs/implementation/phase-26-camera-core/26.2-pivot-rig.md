# 26.2 Pivot Rig Hierarchie

> **Commit:** `feat(phase-26): 26.2 Pivot Rig Hierarchie`

---

## Ziel

Transform-basierte Pivot-Hierarchie für getrennte Rotationsachsen. Ersetzt die direkte Positions-/Rotationsberechnung der alten ThirdPersonCamera.

---

## Hierarchie

```
CameraRoot (Position = Anchor, Inertia offset)
    └── YawPivot (Y-Rotation = horizontale Drehung)
          └── PitchPivot (X-Rotation = vertikale Neigung)
                └── OffsetPivot (localPosition = ShoulderOffset)
                      └── Camera (localPosition.z = -Distance)
```

### Vorteile gegenüber Monolith

- **Yaw/Pitch unabhängig** — keine Quaternion-Kombination, kein Gimbal Lock
- **ShoulderOffset ohne Rotation** — seitlicher Versatz rotiert automatisch mit
- **Distance unabhängig** — Zoom ändert nur Camera.localPosition.z
- **Erweiterbar** — weitere Pivots einfach einfügbar (z.B. für Inertia, Shake)

---

## Implementierung

**`Runtime/Rig/PivotRig.cs`**

```csharp
namespace Wiesenwischer.GameKit.Camera
{
    /// <summary>
    /// Hierarchische Camera-Rig-Struktur mit getrennten Rotationsachsen.
    /// Wird vom CameraBrain pro Frame mit dem finalen CameraState aktualisiert.
    /// </summary>
    public class PivotRig : MonoBehaviour
    {
        // Serialisierte Referenzen (Editor-Setup oder Auto-Create)
        [SerializeField] private Transform _yawPivot;
        [SerializeField] private Transform _pitchPivot;
        [SerializeField] private Transform _offsetPivot;
        [SerializeField] private Transform _cameraTransform;

        // Properties für CameraBrain-Zugriff
        public Transform CameraTransform => _cameraTransform;
        public Transform Root => transform;

        /// <summary>
        /// Wendet den CameraState auf die Hierarchie an.
        /// Aufgerufen vom CameraBrain nach Behaviour-Evaluation.
        /// </summary>
        public void ApplyState(CameraState state, Vector3 anchorPosition)
        {
            // Root folgt dem Anchor
            transform.position = anchorPosition;

            // Yaw = horizontale Rotation um Y-Achse
            _yawPivot.localRotation = Quaternion.Euler(0f, state.Yaw, 0f);

            // Pitch = vertikale Rotation um X-Achse
            _pitchPivot.localRotation = Quaternion.Euler(state.Pitch, 0f, 0f);

            // Shoulder Offset
            _offsetPivot.localPosition = state.ShoulderOffset;

            // Distance = Kamera-Abstand auf -Z
            _cameraTransform.localPosition = new Vector3(0f, 0f, -state.Distance);
        }

        /// <summary>
        /// Erstellt die Pivot-Hierarchie automatisch als Child-Transforms.
        /// </summary>
        public void EnsureHierarchy()
        {
            if (_yawPivot == null)
                _yawPivot = CreateChild(transform, "YawPivot");
            if (_pitchPivot == null)
                _pitchPivot = CreateChild(_yawPivot, "PitchPivot");
            if (_offsetPivot == null)
                _offsetPivot = CreateChild(_pitchPivot, "OffsetPivot");
            if (_cameraTransform == null)
            {
                var cam = GetComponentInChildren<UnityEngine.Camera>();
                if (cam != null)
                    _cameraTransform = cam.transform;
                else
                    _cameraTransform = CreateChild(_offsetPivot, "Camera");
            }
        }

        /// <summary>Welt-Position der Kamera (für Collision etc.).</summary>
        public Vector3 GetCameraWorldPosition()
            => _cameraTransform.position;

        /// <summary>Welt-Rotation der Kamera.</summary>
        public Quaternion GetCameraWorldRotation()
            => _cameraTransform.rotation;

        /// <summary>Forward-Vektor der Kamera in Welt-Space.</summary>
        public Vector3 GetCameraForward()
            => _cameraTransform.forward;

        private static Transform CreateChild(Transform parent, string name)
        {
            var go = new GameObject(name);
            go.transform.SetParent(parent, false);
            go.transform.localPosition = Vector3.zero;
            go.transform.localRotation = Quaternion.identity;
            return go.transform;
        }
    }
}
```

---

## Hinweise

- `EnsureHierarchy()` wird vom CameraBrain in `Awake()` oder vom Editor-Setup aufgerufen
- Die Hierarchie kann im Editor manuell aufgebaut werden (Serialisierte Felder) oder automatisch via `EnsureHierarchy()`
- FOV wird direkt auf die Unity Camera gesetzt (nicht über Pivot)
- In Phase 28 wird ggf. ein CinemachineDriver dazwischengeschaltet
