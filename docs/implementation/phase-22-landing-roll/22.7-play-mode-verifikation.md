# 22.7 Play Mode Verifikation

> Kein eigener Commit/Branch — wird als Teil des letzten Feature-Branches oder direkt auf dem Integration-Branch verifiziert.

---

## Ziel

Alle Landing Roll Features im Play Mode manuell testen und bestätigen.

---

## Test-Szene Vorbereitung

In der `AnimationTestScene` oder `PlaygroundScene` folgende Elemente sicherstellen:

1. **Hohe Plattform** (> HardLandingThreshold Fallhöhe, z.B. 5-10m)
2. **Niedrige Stufe** (< HardLandingThreshold, z.B. 1-2m) — für SoftLanding-Vergleich
3. **Mittlere Höhe** (knapp über HardLandingThreshold) — für Grenzfall-Test
4. **Freie Fläche** zum Rollen (keine Hindernisse direkt unter Plattform)
5. **Steile Rampe** (> MaxSlopeAngle) — für Slide vs. Roll Abgrenzung

---

## Manuelle Test-Checkliste

### Grundfunktion

- [ ] **Hoher Fall + Vorwärts-Input:** Character rollt vorwärts, nahtloser Übergang zu Run
- [ ] **Hoher Fall + Seitlicher Input:** Character rollt in seitliche Richtung
- [ ] **Hoher Fall + Rückwärts-Input:** Character rollt rückwärts
- [ ] **Hoher Fall + Kein Input:** HardLanding (unverändert, Vollstopp)
- [ ] **Niedriger Fall + Input:** SoftLanding (unverändert, kein Roll)

### Animation

- [ ] **Roll-Animation:** Korrekte Roll-Pose wird abgespielt (kein T-Pose)
- [ ] **CrossFade:** Sanfter Übergang Fall → Roll (~0.1s)
- [ ] **CrossFade Roll → Locomotion:** Smooth Übergang nach Roll-Ende
- [ ] **CrossFade Roll → MediumStop:** Smooth Übergang ohne Input
- [ ] **One-Shot:** Animation spielt einmal ab, kein Loop

### Transitions nach Roll

- [ ] **Roll → Run:** Mit Movement-Input → direkt in Run-State
- [ ] **Roll → Sprint:** Sprint gehalten → direkt in Sprint-State nach Roll
- [ ] **Roll → Walk:** Walk-Toggle aktiv → in Walk-State nach Roll
- [ ] **Roll → MediumStop:** Kein Input nach Roll → Abbremsen
- [ ] **Roll → Jump (blockiert):** Jump während Roll wird ignoriert

### Rotation

- [ ] **Sofortige Rotation:** Character dreht sich sofort in Input-Richtung
- [ ] **Omni-Directional:** 8-Wege-Roll funktioniert (alle Richtungen)

### Config-Werte

- [ ] **RollEnabled = false:** Roll wird nie ausgelöst, immer HardLanding
- [ ] **RollSpeedModifier:** Niedrig (0.5) = langsamer Roll, Hoch (2.0) = schneller
- [ ] **RollTriggerMode = ButtonPress:** Roll nur bei DashPressed
- [ ] **RollTriggerMode = MovementInput:** Roll bei Movement-Input (Default)

### Abgrenzung zu anderen States

- [ ] **Slide vs. Roll:** Auf steilem Hang → Slide (nicht Roll), auch bei Movement-Input
- [ ] **SoftLand unverändert:** Niedriger Fall bleibt SoftLand

### Performance

- [ ] **Kein Overhead:** Normaler Fall + Landing hat keinen spürbaren Performance-Einbruch
- [ ] **Kein GC:** Profiler zeigt keine Allocations im Roll-Path

---

## Bekannte Einschränkungen

- Keine I-Frames (Schadens-Immunität) — späteres Combat-Feature
- Keine Stamina-Kosten — späteres Ressourcen-System
- Kein aktiver Dodge (input-getriggert aus Grounded) — separate Spec
- Keine VFX/Particles beim Rollen — späteres Feature
- Keine Sound-Effekte — späteres Feature
- Keine Netzwerk-Synchronisation — Phase 6/7
