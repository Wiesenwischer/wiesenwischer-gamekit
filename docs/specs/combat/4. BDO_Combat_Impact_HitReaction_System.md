# BDO-Style Combat -- Impact & Hit Reaction System (AAA Architecture)

This document summarizes the **Impact / Hit Feedback Stack** --- one of
the most important layers for making action combat feel heavy,
responsive and satisfying, similar to Black Desert Online (BDO).

The focus is NOT on gameplay mechanics alone, but on synchronized
feedback systems that create perceived impact.

------------------------------------------------------------------------

# 1. What Impact Really Means

Impact is NOT simply applying damage.

Instead, impact is a synchronized feedback pipeline:

HitDetected ↓ Impact Pipeline ├ Hit Stop ├ Animation Reaction ├ Camera
Response ├ Audio Feedback ├ VFX Response ├ Motion Feedback └ State
Effects

The goal:

-   Increase perceived weight
-   Improve responsiveness
-   Reinforce player feedback

------------------------------------------------------------------------

# 2. Hit Stop (CRITICAL)

## Definition

A very short time freeze applied when a hit connects.

Typical duration:

0.03 -- 0.06 seconds

Targets affected:

-   Attacker
-   Target
-   Sometimes camera

------------------------------------------------------------------------

## Why Hit Stop Works

-   Brain perceives contact
-   Attack feels heavier
-   Timing clarity increases

Without hit stop:

-   Combat feels floaty
-   Hits lack presence

------------------------------------------------------------------------

# 3. Animation Reaction Layer

Enemy reactions provide immediate feedback.

## Soft Reactions

-   small flinch
-   subtle recoil
-   no gameplay state change

## Hard Reactions

-   stagger
-   knockback
-   stun

Important:

Not every hit needs hard CC --- visual feedback alone often suffices.

------------------------------------------------------------------------

# 4. Camera Feedback

Camera should respond subtly:

Possible effects:

-   micro shake
-   slight FOV pulse
-   tiny forward push

IMPORTANT:

Avoid excessive screen shake.

AAA systems use very subtle adjustments.

------------------------------------------------------------------------

# 5. Audio Feedback

Sound strongly influences perceived impact.

Recommended structure:

-   attack swing sound
-   hit transient layer
-   material-specific impact sound (optional)

Audio timing must align exactly with hit frame.

------------------------------------------------------------------------

# 6. Visual Effects (VFX)

Purpose:

-   reinforce contact visually

Common examples:

-   sparks
-   slash trails
-   dust bursts

Avoid overloading with excessive particles.

------------------------------------------------------------------------

# 7. Motion Feedback

Both attacker and target should react physically.

Attacker:

-   slight recoil or follow-through adjustment

Target:

-   small impulse
-   micro displacement

This reinforces momentum.

------------------------------------------------------------------------

# 8. Impact Pipeline Architecture

## Event Flow

HitDetected ↓ Create ImpactEvent ↓ ImpactSystem distributes to listeners

------------------------------------------------------------------------

## ImpactEvent Example

ImpactEvent { position normal force damageType attackTags }

------------------------------------------------------------------------

# 9. Modular Listener Systems

Each system reacts independently:

-   HitStopSystem
-   CameraSystem
-   AudioSystem
-   AnimationReactionSystem
-   VFXSystem
-   MotionFeedbackSystem

Advantages:

-   Modular design
-   Easy iteration
-   New feedback layers without rewriting combat logic

------------------------------------------------------------------------

# 10. Impact Strength Scaling

Impact intensity may depend on:

-   weapon weight
-   attack phase
-   relative velocity
-   attacker momentum
-   target state

Example:

Heavy attack:

-   stronger hit stop
-   stronger camera response
-   larger motion impulse

------------------------------------------------------------------------

# 11. Common Beginner Mistakes

❌ Only applying damage without feedback

❌ Overusing screen shake

❌ Using animation reaction alone

❌ Not synchronizing systems
